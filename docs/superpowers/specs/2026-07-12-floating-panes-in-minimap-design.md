# 設計仕様: フローティングペインの表示と選択（#110）

> 内部計画ドキュメント。`docs/design.md` と同じく日本語で記述する（本リポジトリの
> 計画ドキュメントの慣例）。コード識別子・config キー・zellij 型名は原語のまま。

## 1. 動機と2つの要求

現在ミニマップは **タイル(tiled)** レイアウトしか描かない。`is_tiled_terminal` が
`is_plugin || is_floating || is_suppressed` を落とす（`src/projection.rs:30-32`）ため、
**フローティングペインはバー上で不可視** — 存在すら見えず、ホイールwalk (#80) や
クリックフォーカス (#74) も到達しない。`docs/design.md:367` が「floating pane の別レイヤー
描画」を先送り事項として挙げていた。本仕様がそれを具体化する。

要求は2つ:

1. **フローティングペインを tabmap に表示する。** タブがフロートを持つこと、そして何個
   持つかが一目で分かるようにする。
2. **フロート層が非表示でも、フロートペインを選択可能にする。** 層をトグルオフすると
   ペインは存在するが画面上の footprint を持たない。それでも狙える UI アフォーダンスが要る。

## 2. 調査結果（zellij-tile 0.44.3 の型）

オープンクエスチョンの多くは型定義で既に答えが出ている。

### `TabInfo`（必要なシグナルが揃っている）

- `are_floating_panes_visible: bool` — フロート層が表示中か（**タブ単位**）。表示/非表示で
  描画をB/A切替する土台。
- `selectable_floating_panes_count: usize` — フロートペイン数を直接取得できる。

### `PaneInfo`

- `is_floating: bool` — フロート判別。
- `pane_x/pane_y/pane_columns/pane_rows: usize` — タブ相対のセル座標。表示中フロートの
  グラフィカル描画に使える（タイルと同じ座標系）。
- **`is_pinned` フィールドは存在しない**（0.44.3）。したがってペイン単位で pin 状態は
  読めない。描画・判断は「タブ単位の `are_floating_panes_visible`」と「ペイン集合」だけに
  キーする。

## 3. 設計原則

### 3.1 バーは zellij 状態の「純投影」

プラグインはフロートを自分で隠す/出すことはしない。毎フレーム、zellij のスナップショット
（`are_floating_panes_visible` ＋ 報告されたペイン集合）から描き直すだけ。

帰結: **「フォーカスが外れたら自動でまた隠す」ロジックを自前で書かない。** zellij が
「unpinned フロートはフォーカスを失うと隠す」挙動なら、フォーカス移動の結果届く次の
`TabUpdate`/`PaneUpdate` で `are_floating_panes_visible` が false に倒れ、そのタブの描画が
オーバーレイ→チップに自動で切り替わる。手を出さなくてもバーが追従する。

補強: zellij には「特定の1フロートだけを隠す」アクションが無い（あるのは層トグルと pin）。
後始末を自前でやるのは筋が悪く、ネイティブ挙動に乗るのが正解。

### 3.2 タイル投影にフロートを混ぜない

タイル層の bounding-box 正規化にフロートを混入させるとタイルのトポロジが歪む。タイル投影は
現状のまま維持し、フロートは**同じ bbox 変換を通した別レイヤー**として合成する。

## 4. 決定事項（ブレインストーミングで確定）

| 項目 | 決定 |
|---|---|
| デフォルト挙動 | **デフォルト ON**。標準でフロートを描画、config で opt-out。フロート0個のタブは描画一切不変。 |
| 非表示フロートの表現 | **個別チップ**。1ペイン=1セルのヒット対象。issue #2 を満たす。 |
| 対象ペイン | **フローティングのみ**。suppressed は今回対象外。 |

## 5. 振る舞い（タブ単位で切替）

`are_floating_panes_visible` はタブ単位のフラグで、1タブ内のフロートは全部まとめて
表示/非表示（層全体が対象で、ペイン個別ではない）。したがって、描画は以下のようになる:

| タブの状態 | 描画 |
|---|---|
| フロート表示中 (`are_floating_panes_visible == true`) | 各フロートをタイルミニマップ上に**グラフィカルにオーバーレイ** |
| フロート非表示 | 各フロートを右下隅に**個別チップ**（1ペイン=1セル） |
| フロート0個 | 何も変わらない（現状のまま） |

対象は**全グリッドラング（L0–L2）タブ**。既存 #74 と同じく非アクティブタブのフロートも
クリック可能に。L3/L4 の狭いラングは描画余地がないため対象外（チップも描かない）。

## 6. レンダリング方針

### 6.1 オーバーレイ（表示中フロート）

各フロートの `pane_x/y/w/h` を、タイルと同じ bbox 変換で block-local ピクセルに写像し、
タイルグリッドの**上に**塗る。フロートは実際にタイルの上に浮くので、上描きは空間的に忠実。

- タイル塗りと区別するため薄い 1px ボーダー(リング)を付ける。
- 色はフロート id をキーにパレットから（タイルと同じ id→色の仕組み）。
- ラベルは付けない（面積が小さい）。色＋ボーダーのみ。
- **v1 はフォーカス状態に依らず素ボーダーのみ**。アクティブタブのフォーカス中フロートを
  focus ring（#59 準拠）で強調するのは follow-up に送る（Task 10 の方針どおり。フロートは
  一度に1つしかフォーカスされず、ボーダーだけでも十分に区別できるため v1 では割り切る）。
- フロート座標がタイル bbox の外に出る場合はブロックにクランプ（稀）。

### 6.2 チップ（非表示フロート）

最終行の**右下隅**に、フロート1つにつき1セルのマーカー（例 `◲`）を id 順に配置。

- 各セルの列を**ヒット領域として記録**（#74 の `tab_panes` と並列の仕組み）。
- 入りきらない時は末尾を `+k` に畳む（#113 以降、`+k` クリックは畳まれた先頭フロートを
  reveal＋フォーカス。reveal で層全体が可視化されるため、残りはオーバーレイ上で個別に
  クリック可能になる → silent-truncation にはしない）。
- アクティブは明色、非アクティブは #59 準拠でミュート。
- バッジ(`⌘N`)は左上、チップは右下で衝突しない。

### 6.3 モジュール境界

- 新規 `src/floating.rs`（dependency-free）: block ジオメトリ＋フロートデータを入力に、
  オーバーレイのピクセル写像とチップセル＋ヒット領域を生成する純関数群。renderer と同じ
  「zellij 型に依存しない」discipline を守り、`cargo test --lib` でネイティブテスト。

## 7. インタラクション

### 7.1 クリック解決順

フロート（チップ/オーバーレイ）→ タイルペイン → タブ切替。同一セルがタイルとオーバーレイ
両方に属する場合は**フロート優先**（実際に上に浮いているため）。

### 7.2 フォーカスプリミティブ

`focus_terminal_pane(id, should_float_if_hidden=true, false)`。

- 非表示チップを押す → **表示＋フォーカス**（reveal は `should_float_if_hidden=true` で無料）。
- 表示中フロートは既に見えているので実質フォーカスのみ（`true` を渡しても無害）。
- 新規パーミッション不要（既存の `ChangeApplicationState` で足りる）。

### 7.3 ホイール walk（#80 pane モード）

各タブのタイルペインの**後ろ**にフロート id を連結して walk に含める。方針は P0 スパイクの
「自動 hide」結果で確定する2択:

- **自動 hide を確認できた場合（推奨採用）:** walk にフロートを含める。非表示フロートに
  ホイールで乗る＝reveal＋フォーカス。次の一撃で隣のタイルに移ると zellij が自動で層を畳む。
  reveal は「フォーカス中だけ」の一時表示になり後腐れなし。
- **自動 hide しない場合:** walk はタイル＋表示中フロートのみに絞る。非表示フロートは
  ホイールでは開かず、チップのクリックだけが reveal の入口（明示操作なので開きっぱなしでも許容）。

## 8. Config

他の enum（`ScrollMode`/`GradientMode`）に倣い `src/config.rs` + 描画層に `FloatingMode` を新設。

- config キー `floating`:
  - `"hybrid"`（**デフォルト**、§5 のB/A挙動）
  - `"off"`（現状の見た目を完全維持）
- パースは total（不正値は default にフォールバック、既存 config と同じ discipline）。
- 新規パーミッションは不要。

## 9. P0 スパイク（着手前・必須。`docs/design.md:378` §9 のスパイク先行方針に準拠）

eprintln オラクル harness（`.claude/rules/zellij-plugin-development.md` §4）で実機裏取り。
結果が以降フェーズを gate する。

1. **非表示フロートが `PaneManifest` に id 付きで残るか。** → 個別チップ選択の可否を決める
   最重要点。残らなければチップはカウントバッジに縮退（issue #2 未達 → 要再判断）。
2. **表示中フロートの `pane_x/y/w/h` が期待どおりタイル座標系で取れるか。** → オーバーレイ
   描画の前提。
3. **フロート→タイルにフォーカスを移した時、`are_floating_panes_visible` が自動で false に
   なるか。** → §7.3 ホイール walk の後始末が「無料」か否かを決める。

## 10. 段階的実装（各フェーズ独立してテスト可能）

- **P0 スパイク:** §9 の裏取り（残りを gate）。eprintln オラクルは commit しない。
- **P1 データ + config:** `projection.rs` にフロート抽出関数を追加（表示中は id+rect+focused、
  非表示は id リスト）、`FloatingMode` config 配線。描画はまだ。ユニットテスト付き。
- **P2 チップ + 非表示選択:** `floating.rs` のチップ生成、`minimap`/`tab_block` へ配線、
  `lib.rs` にヒットテストとクリックで reveal＋フォーカス。**issue #2 達成**。
- **P3 表示中オーバーレイ:** `floating.rs` のオーバーレイ写像、グラフィカル合成、クリック/
  リング。**issue #1 達成**。
- **P4 ホイール walk:** `pane_focus_order` にフロート連結（§7.3、P0-3 の結果で最終確定）。

## 11. リスクと緩和

| リスク | 影響 | 緩和 |
|---|---|---|
| 非表示フロートが manifest に id 付きで残らない | 個別チップ選択が不可能 | P0-1 で最優先確認。ダメなら count バッジへ縮退し #2 を再設計。 |
| `are_floating_panes_visible` が自動 hide しない | ホイールで開いた層が残る | P0-3 で確認。ダメなら §7.3 の後者（walk はタイル＋表示中のみ）を採用。 |
| 狭い block にチップが入りきらない | 一部フロートが選べない | `+k` 畳みで明示（silent truncation を避ける）。#113 以降は `+k` クリック→reveal→オーバーレイ個別クリックの経路があり、全フロートが最大2クリックで到達可能。 |
| pin 状態が読めない（0.44.3） | pinned フロートの区別描画ができない | 今回はタブ単位の可視フラグだけにキーし、pin 区別はスコープ外（将来 zellij 更新時に再検討）。 |

## 12. スコープ外 / 先送り

- suppressed / background ペインの表示（§4 決定どおり今回は対象外）。
- pinned フロートの区別スタイル（0.44.3 では読めない）。
- オーバーレイ内のフロートラベル（面積不足）。
