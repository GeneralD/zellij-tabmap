# 表示中フロートによるタイルラベルのオクルージョン＋truncate cue 設計

**関連**: #110（フロートを minimap に描く）の追加改善。PR #112 に上乗せ。

## 背景 / 目的

現在、表示中フロートは `minimap::render` の二重ループで `float_px`（フロート塗り）
より **前に** ラベル（`OverlayCell::Glyph`）を描き `continue` するため、タイル pane の
ラベル文字がフロートの **上** に乗って見える（デモで "cargo"/"test" がフロートに重なって
読める状態）。本来フロートは上に浮いた窓なので、下のラベルを **覆い隠す** のが自然で
「面白い」。ただし覆われた文字が単に消えると "carg"/"est" が実 pane 名と誤読されるため、
**切れた境界に手掛かり**を出す。

## 決定事項（承認済み）

1. **オクルージョン**: 表示中フロートが覆うセルでは下のタイルラベルを描かない。
   フロート塗りが優先。
2. **truncate cue = `…` ＋ 控えめな影**: ラベルがフロートの下へ続く境界セルは、
   元文字の代わりに `…`（U+2026, 表示幅1）で描き、そのセル背景を黒方向へ **25%**
   ブレンドした控えめなドロップシャドウを敷く。`…` グリフは影付き背景の上に重ねて描く。
3. `…` は既存の幅 truncate（`title::summarize` の `実装…`）と同じグリフ＝一貫性。

## 詳細仕様

### 覆い（occlusion）判定

- セル `(tr, c)` が **フロート下**: `float_grid` が画素 `(2·tr, c)` または `(2·tr+1, c)`
  のいずれかを覆う（半ブロックの上下どちらかが覆われていれば覆い扱い）。
- ラベル `Glyph` セルがフロート下なら描かず、既存の `float_px`／タイル塗りへフォールする
  （覆われた画素はフロート色、未覆いの画素はタイル塗り）。
- `Continuation` セルもフロート下なら「何も出さず `continue`」ではなくフォールさせる
  （下のフロート／塗りを出す）。

### 境界（truncate）判定と cue

- 描画される（＝フロート下でない）ラベル `Glyph` セル `(tr, c)` が **境界** となるのは、
  グリフの表示幅を跨いだ隣セルが「フロート下 かつ その位置に同じラベルの
  `Glyph`/`Continuation` が存在」する場合。
  - 右境界: `x = c + width(glyph)`。`has_label(x) && float_over(x)`。
  - 左境界: `c > 0` かつ `has_label(c-1) && float_over(c-1)`。
  - `has_label(x)` = `overlay[tr·pw + x]` が `Glyph`/`Continuation`。
- **誤爆しない**: ラベルがフロート手前で自然に終わっているだけ（隣がフロートでも
  そこにラベルが無い）なら境界にならず cue は出ない。「本当に下へ続く時だけ `…`」。
- 境界セルは `…`（幅1）で描画し、背景 = `mixed(label_fill, (0,0,0), 25)`。
  前景色・dim は通常ラベルと同じ。`…` は本来の文字ではないので太字強調は付けない。

### 幅コントラクト（重要な端ケース）

- 境界が **全角グリフ**（`Glyph`＋`Continuation` で2桁）に当たる場合、`…`（幅1）へ
  置換すると1桁になる。続く `Continuation` セルがフロート下（=覆い）なら、そのセルは
  フォールしてフロート塗りが1桁埋めるので、行幅は保たれる。実装・テストで全角境界を
  必ず検証する。

### スコープ / 非目標

- アクティブ／非アクティブ両タブに適用（`…` は既存の非アクティブ dim をそのまま継承）。
- `floating = hybrid` のときのみ（`off` は従来通り、フロート自体を描かない）。
- **フロート自身のタイトル描画はしない**（今回の要望外・YAGNI）。ボーダー／塗りは現状維持。
- 隠しフロートのチップ、非表示フロートのクリック等 #110 既存挙動は不変。

## 実装方針（テスト容易性）

- 判定を render ループに埋め込まず、zellij 型を持たない **純粋ヘルパー**へ切り出し、
  `cargo test --lib` で単体テスト（既存の renderer 分離方針どおり）。
  - 例: `label_over_float(overlay, float_grid, pw, tr, c) -> LabelDraw`
    （`Char` / `Ellipsis`（=`…`+shadow） / `Occluded`）。
- 影ブレンド率は名前付き定数（例 `LABEL_SHADOW_BLEND = 25`）。

## テスト観点

1. フロート下のラベルセルは隠れ、フロート色が出る。
2. 境界セルは `…` になり、背景が暗くなる（shadow）。
3. フロートに接するだけで下に続かないラベルは `…` にならない（誤爆なし）。
4. 全角グリフ境界でも行幅が保たれる。
5. フロート無し／隠しフロート経路は byte-identical（既存
   `render_without_floats_is_byte_identical_to_none` 等を維持）。
6. 既存の幅コントラクト（各行の表示幅）を維持。
