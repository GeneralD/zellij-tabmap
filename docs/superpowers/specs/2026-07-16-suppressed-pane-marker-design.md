# 設計仕様: suppressed ペインの存在マーカー（#118）

> 内部計画ドキュメント。`docs/design.md` と同じく日本語で記述する（本リポジトリの
> 計画ドキュメントの慣例）。コード識別子・config キー・zellij 型名は原語のまま。

## 1. 動機

フローティングペインの表示（#110、PR #112・focus ring の #116 いずれもマージ済み）は
**suppressed（背景）ペインを明示的にスコープ外**とした（floating 設計 §4 決定 / §12）。
現状 `is_tiled_terminal` / `is_floating_terminal` が両方とも `is_suppressed` を落とす
（`src/projection.rs:31,40`）ため、suppressed ペインはバー上に痕跡を残さない。

zellij の suppressed ペインは、それを置き換えたペイン（以下 **カバーペイン**）の
**真下・同一スロット**に隠れる存在で、たいてい一時的（典型例は edit-scrollback が
`$EDITOR` を元ペインの上に開き、元ペインを suppress するケース）。「別の矩形として
描く」場所が無く、カバーペインとジオメトリが重なる。

本仕様は、この suppressed ペインの **存在**を、カバーペインの隅の小さなマーカーで
知らせる（awareness）ことだけを目的とする。レイアウトの完全表現でも、復帰/フォーカス
操作でもない。

## 2. 調査結果（zellij-tile 0.44.3）

- `PaneInfo.is_suppressed` は読める。suppressed ペインは manifest に**残る**（現状は
  こちらが除外しているだけ）。
- **未確認（P0 スパイク対象、§7）**: suppressed ペインの `pane_x/y/columns/rows` が
  有効値で入るか、そしてカバーペインと**同一/内包**の関係になるか。#18 の隠れフロート
  調査（`.claude/rules/zellij-plugin-development.md` #18）と同じ手法で実測して確定する。
- 追加ホストコールなし＝**新規 zellij 権限は不要**（既存の `ReadApplicationState` の範囲、
  floating 設計 §15・rule #15 のトラップに触れない）。

## 3. 決定事項（ブレインストーミングで確定）

| 論点 | 決定 |
|---|---|
| 目的 | **存在の気づき（awareness）のみ**。復帰/フォーカス操作・完全レイアウト表現はしない。 |
| 置き場所 | **カバーペインの右下隅**に1セルのマーカー（案 A）。あなたのプレビュー通り「どのペインの裏か」を示す。 |
| 範囲 | **アクティブタブのみ**。非アクティブタブには出さない（一時的な suppress のノイズを避ける）。 |
| デフォルト | **常時 ON**。config キーの追加なし（権限不要なので #15 の凍結リスクと無関係）。 |
| 操作 | **なし（描画のみ）**。クリックハンドラを増やさない。 |
| フォールバック | カバー特定が spike で不安定なら、**案 B（アクティブタブブロックの右下隅に件数マーカー）** に切替。 |

## 4. 設計

### 4.1 データ / projection

- 既存のタイルフィルタ `is_tiled_terminal` は **一切変えない**（#110 のバイト同一性を
  保つ）。
- `projection.rs` に純粋関数を追加し、**アクティブタブの suppressed ペインを
  `(id, rect)` で抽出**する。`is_floating_terminal` / `project_floating` と同じ作法・
  同じ純粋関数スタイル（zellij 型を混ぜない、`cargo test --lib` で単体テスト可能）。
  - フィルタ述語: `pane.is_suppressed && !pane.is_plugin`（プラグイン由来の suppress は
    除外。floating との排他は spike で確認するが、基本は tiled スロットのカバーを狙う）。

### 4.2 カバー特定（案 A の肝・spike 対象）

- 各 suppressed ペインの rect を、可視タイルペインの矩形集合と突き合わせ、**同一スロットを
  占める可視ペイン=カバーペイン**を特定する（同一原点＋同一サイズ、または内包）。
- カバーが特定できた suppressed ペインについて、その**カバーペインの id** にマーカーを
  1つ立てる（同じカバーの裏に複数 suppress されていても **presence のみ**＝1つ点灯。
  件数は出さない。awareness には十分）。
- 依存ゼロの純粋ヘルパーとして実装し、`(suppressed rects, visible tiled rects) →
  マーカーを立てるカバー id 集合` を単体テストする。

### 4.3 レンダリング

- カバーペインの**右下1セル**に、隠れフロートチップ `◲`（`CHIP_GLYPH`）とは**別グリフ**を
  描く。既定は **`◳`**（プレビュー通り）。隠れフロートチップとは *グリフの回転＋描画場所*
  （フロートチップ=タブブロック隅 / suppressed マーカー=個別ペイン隅）で区別し、
  「隅の隠れ物マーカー」という家族感は保つ。グリフは名前付き定数にして後から差し替え可能に
  する。
- **precedence**: 隠れフロートチップ（タブブロック右下に予約セルを持つ、#110 / #113）と
  セルが競合し得る。マーカーはチップの**予約セルを踏まない**。カバーペインが小さすぎて
  マーカーの1セルが取れない場合は**描かない**（潰れた描画をしない）。
- 塗りは既存のペイン fill の上に重ねる。ミニマップは依存ゼロのレンダラ（`src/minimap.rs`、
  zellij 型なし）なので、この描画も純粋テストで検証する（rule #7 の paint 側検証）。

## 5. インタラクション

なし。マーカーは**描画のみ**。クリック解決順（floating 設計 §7.1、`src/router.rs`）は
変更しない。将来「復帰/フォーカス」に踏み込むなら別 issue（本 #118 のスコープ外）。

## 6. Config / 権限

- **config キーの追加なし**（常時 ON）。
- **新規 zellij 権限なし**（read-only、既存グラント内）。既存ユーザーの見た目・権限グラントを
  一切変えない（#15 の「新権限でバー凍結」トラップに触れない）。

## 7. P0 スパイク（着手前・必須）

floating 設計 §9 と同じスパイク先行方針に従い、実装前に **eprintln オラクル + expect PTY**
（rule #4）で以下を実測する:

1. edit-scrollback（`$EDITOR` を開く）で suppressed ペインを1枚作る。
2. `PaneManifest` を eprintln で観測し、suppressed ペインの:
   - `is_suppressed` フラグ
   - `pane_x/y/columns/rows`（有効値か / カバーと同一・内包か）
   を確認する。
3. **判定**:
   - カバー特定が安定 → **案 A**（カバーペイン隅マーカー）で実装。
   - 不安定 / ジオメトリが読めない → **案 B**（タブブロック隅に件数マーカー）へフォールバックし、
    この仕様の §4.2〜§4.3 を差し替える。

### スパイク結果（実施済み 2026-07-16、zellij 0.44.3）→ **案 A 確定**

隔離セッション `tabmap-spike118` ＋ `DBG118` オラクル ＋ expect PTY で実機裏取り済み。
`new-pane --in-place`（置換ペインを *suspend* する）と `edit-scrollback` で terminal の
suppressed を生成し観測:

- terminal の suppressed ペインは manifest に出る: `id=0 suppressed=true plugin=false
  x=0 y=3 w=120 h=37`（有効な幾何）。
- カバー（置換ペイン `id=1` focused `title="EDITING SCROLLBACK"`）は**同一 rect**
  `x=0 y=3 w=120 h=37` → suppressed rect はカバー rect に**内包**（＝ `covers()` 成立）。
- suppressed な**プラグイン**ペインも存在（attention: `suppressed=true plugin=true
  x=30 y=10 w=60 h=20`）。`is_suppressed && !is_plugin` 述語は terminal を**含み**プラグインを
  **除外**することを確認。
- タイルペインは空間を重複なく分割するため、suppressed rect を内包するタイルはカバー1枚のみ
  → `cover_ids` はカバーだけをマークする。**案 A で実装する。**

（プラグインパネルは `dump-screen` で読めない（rule #7）ため、データ/投影はオラクルで、
paint はレンダラ単体テストで、という #110 と同じ二分割検証を踏襲する。）

## 8. リスクと緩和

| リスク | 影響 | 緩和 |
|---|---|---|
| suppressed のジオメトリが読めない/重ならない（未確認） | 案 A のカバー特定不能 | §7 スパイクで先に判定 → 案 B にフォールバック |
| 一時的な suppress の点滅ノイズ | マーカーがちらつく | アクティブタブのみに限定（§3）。件数は出さず presence のみ |
| フロートチップとの誤認 | 「隠れフロート」と「裏の suppressed」を混同 | 別グリフ（`◳`）＋描画場所（個別ペイン隅 vs タブ隅）で区別 |
| チップ予約セルとの競合 | 描画が重なる/潰れる | チップの予約セルを踏まない。1セル取れなければ描かない（§4.3） |

## 9. スコープ外 / 先送り

- suppressed ペインの**完全レイアウト表現**（ハッチ/減色を面で重ねる案 B の面表現）。
- バーからの**復帰/フォーカス操作**（zellij 側 API の可否確認と、恐らく新権限が必要）。
- **非アクティブタブ**での表示。
- 裏の**枚数表示**（presence のみで割り切る）。
